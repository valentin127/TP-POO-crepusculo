Sistema de Gestión de Crepúsculo

Consigna: el trabajo práctico tiene un tiempo de entrega de 10 semanas. El equipo de trabajos
prácticos deberá implementar un sistema de Crepúsculo utilizando los conceptos aprendidos
en clase. Se deberá entregar un archivo “.zip” con la carpeta del proyecto. Además, cada
integrante del equipo debe tener pleno conocimiento para realizar una defensa oral.

Criterio de aprobación: el programa debe tener bien la estructura de clases y debe funcionar
correctamente. Si la estructura no cumple la consigna, está desaprobado, aunque funcione.
Si el programa no funciona correctamente, está desaprobado, aunque la estructura esté bien
hecha.

Requisitos del sistema:

1) Clase Vampiro (abstracta)
a) Atributos: nombre, fuerza, velocidad, hambre, colorOjos.
b) Métodos: comer, habilidadEspecial.

2) Clases que extienden de Vampiro (clases hijas)
a) Cada tipo de Vampiro: recienConvertido, maduro, adulto.

3) Composición y Agregación
a) Incluir otros componentes, por ejemplo: Volturi (conjunto de vampiros
organizados), Animales (alimento para los vampiros).

4) Interfaz
a) Incluir otros métodos, por ejemplo: comerAnimal (algunos vampiros no
comen humanos).
Requisitos de funcionamiento:

5) Menú interactivo que permita realizar las siguientes operaciones:
a) Admitir Vampiro: permite añadir un vampiro al clan.
b) Expulsar Vampiro: permite eliminar a un vampiro del clan.
c) Comer animal: se consume algún animal silvestre.
d) Listar Vampiros: el sistema mostrará todos los vampiros de un clan y sus
características.
e) Obtener Vampiro más apto: muestra en pantalla la información del vampiro
que más fuerza y velocidad tiene, pero con menor hambre.
Recuerde que este es un esquema. Es válido sumar más atributos, métodos o clases,
siempre y cuando tengan una razón de ser.
